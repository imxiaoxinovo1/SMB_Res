# 数据泄露问题说明和模型版本对比

## 问题发现

在分析 RandomForest 模型结果时，发现 **AAR（Accumulation Area Ratio，累积区面积比）** 特征的重要性高达 **0.541**，远超其他所有特征。

经过仔细分析和查阅 WGMS FoG 数据库文档（`readme.pdf`），发现：

### AAR 的定义
> **AAR = Accumulation area / Total glacier area**
>
> 累积区面积比 = 累积区面积 / 冰川总面积

### 为什么这是数据泄露？

AAR 不是物质平衡的**驱动因素**，而是物质平衡的**结果**：

1. **物理关系**：
   - 年物质平衡 > 0 → 累积区扩大 → AAR 增大
   - 年物质平衡 < 0 → 消融区扩大 → AAR 减小

2. **因果关系**：
   ```
   气候条件 → 物质平衡 → AAR/ELA

   而不是：
   AAR/ELA → 物质平衡  ❌（这是错误的因果方向）
   ```

3. **数据泄露机制**：
   - 模型通过 AAR 直接"看到"了物质平衡的结果
   - 这导致虚高的预测性能（R² = 0.676）
   - 但在实际应用中，我们无法提前知道未来的 AAR

### ELA 也有同样的问题

**ELA（Equilibrium Line Altitude，平衡线高度）** 也是物质平衡的结果：
- ELA 是年物质平衡为 0 的高度
- 它直接由物质平衡的垂直分布决定
- 同样不能作为预测物质平衡的特征

## 解决方案

### 创建了两个版本的模型

#### 版本 1：包含数据泄露（原始版本）
- **文件**：`RF_model_LOOCV.py`
- **特征**：包含 AAR, ELA, AREA 等所有几何特征
- **性能**：R² = 0.6763 ⭐ 但有数据泄露！
- **用途**：仅用于理解数据泄露的影响

#### 版本 2：无数据泄露（推荐版本）⭐
- **文件**：`RF_model_LOOCV_no_leakage.py`
- **特征**：排除 AAR 和 ELA，仅保留：
  - 气候变量（年度 + 夏季）
  - 地理位置（纬度、经度）
  - 冰川面积（AREA）- 可以提前知道
  - 高程范围（LOWER_BOUND, UPPER_BOUND）- 可以提前知道
  - 时间（YEAR）
- **性能**：待运行查看真实性能
- **用途**：用于实际预测和研究

## 哪些特征可以保留？

### ✅ 可以保留的特征（物理驱动因素）

1. **气候变量**（全部）
   - 温度、降水、降雪、融雪、辐射、风速等
   - 这些是物质平衡的**驱动因素**
   - 可以从气候数据中提前获得

2. **地理位置**
   - LATITUDE（纬度）
   - LONGITUDE（经度）
   - 这些是冰川的固有属性

3. **冰川几何**（部分）
   - **AREA**（面积）- 可以从遥感数据提前获得
   - **LOWER_BOUND**（下界高程）- 地形属性
   - **UPPER_BOUND**（上界高程）- 地形属性

4. **时间**
   - **YEAR** - 捕捉长期气候变化趋势

### ❌ 必须排除的特征（数据泄露）

1. **AAR**（Accumulation Area Ratio）
   - 原因：是物质平衡的结果
   - 只有在观测当年结束后才能知道

2. **ELA**（Equilibrium Line Altitude）
   - 原因：是物质平衡的结果
   - 由物质平衡的垂直分布决定

## 文献参考

在冰川物质平衡建模研究中，常见的做法：

### ✅ 正确的做法
- 使用气候数据（温度、降水、辐射等）
- 使用地形数据（高程、坡度、坡向）
- 使用冰川几何（面积、长度）

### ❌ 错误的做法
- 使用 AAR 或 ELA 作为预测变量
- 使用当年的累积/消融区面积
- 使用任何需要物质平衡观测才能获得的变量

## 如何验证是否有数据泄露？

### 检查清单

1. **逻辑检查**：
   - 这个特征能在预测时提前获得吗？
   - 这个特征是因还是果？

2. **重要性检查**：
   - 某个特征的重要性异常高（>0.3）？
   - 物理上合理吗？

3. **性能检查**：
   - R² 异常高（>0.8）？
   - 可能有问题

4. **时间检查**：
   - 特征值是否在预测时刻可用？
   - 还是需要等到年末观测？
